## 리엑트의 역사:
자바 스크립트는 사실 라이브 스크립트라는 이름의 언어였다! 하지만 어른의 사정으로 잘나가던 자바의 이름을 따서 자바 스크립트가 되어버렸다.   
자바 스크립트의 시작은 불행했다. 그러나 2004년 자바 스크립트에게 격변기가 온다   

#### 자바 스크립트의 원래 목적:  
스크립트 언어라고 부르는 언어들의 특징: 특정 어플리케이션에서 동작하는 언어들. 범용적이기보단 특수성이 강조된다. 그리고 자바 스크립트(당시의 라이브 스크립트)는  
브라우저에 특화된 찌끄만한 언어가 필요했기에 개발되었다 한다. 

기본적으로 싱글 스레드인점, 인터프리터 방식으로 동작하는 점.   

자바 스크립트가 가장 기본적으로 제공하는 API: Document OM (DOM) && Window OB (BOM)  
자바 스크립트는 이들 안에서 변수/함수 등을 이용해 가공하여 브라우저에 적용한다.  

##### Ajax:
자바 스크립트는 Ajax 이후로 엄청난 격변을 맞이한다. Ajax는 마이크로 소프트가 만듦. 
어플리케이션으로서 운영체제와 연동하는 방법 중 화면의 이동없이 처리가 가능하게 해주는 AJax의 등장은 엄청난 이점을 가져다 줬다.  

이후 HTML5의 등장 이후부턴 웹이 모바일에 대한 지원, 다양한 기술(웹소켓 등)이 지원받기 시작하면서 또 다시 격변이 시작되었다.  
이전에 웹은 단순한 Document였기에 내용이 중요해서 해시베이스(#) 를 많이 이용해 잡근하는 경우가 많았다.  
이후 Ajax와 함깨 웹 페이지의 시대가 온다.    
그리고 HTML5의 등장 이후로 부턴 웹 어플리케이션(Web App)의 시작이 온다.   
그리고 현재, PWA(Progressive Web Application)의 시대가 오는 것 같다는게 강사님의 의견.   

### Library NOT Framework:
리엑트는 라이브러리 이지 프레임워크가 아니다. 그 둘의 차이는 내가 맞춰서 사용을 해야하나, 아니면 내 취향대로 사용해야 하는가이다.   
라이브러리는 기능에 포커스를 맞추고, 프레임 워크는 개발 전체 프로세스에 포커스를 맞춘다.   
리엑트는 브라우저에 포커스를 맞춘 라이브러리라고 한다. 리엑트는 MVVM(Model View View Model) 모델을 채용한 라이브러리라고 한다.

## Web App이란?
리엑트는 웹 엡을 추구한다. 그렇다며 웹 엡은 무슨 의미일까? 앱 처럼 동작하는 웹이다. 웹과 앱의 동작시 가장 큰 차이는 뭘까? 채감 속도이다. 빠른 속도를 지향하려면, 원래 있던 건 그대로 두고  
필요한 것만 바로바로 가져다 쓸수 있게 만드는 것이다.  
물론 다운사이드도 존재한다. 앱은 초기 로딩이 굉장히 느리다. 모든 걸 다 가져와야 하기 때문이다. 그렇기에 지금의 웹 앱은 약간 과도기 적인 성향을 띈다는게 강사님의 의견   

이러한 구조를 자바 스크립트로 만들기는 상당히 어렵다.. 에초에 import 개념도 없고 각기 다른 자바 스크립트 파일간의 변수/함수 공유가 가능하려면 자바의 static 개념으로 만들어야만 한다.   
당연히 이는 굉장히 비효율적이다. 그렇기에 리엑트가 탄생했다 할 수 있다.  


JQuery를 사용할때 HTML코드에 이벤트/함수등을 걸지 말라하는 이유는 HTML은 Model이긴 때문이다. 그렇기 때문에 View + Controller를 담당해 줄 수 있는 JavaScript 를 이용하는 것이  
더 적절한 방법이라고 할 수 있다.   
순수해서 얻는 이득이 뭘까? 데이터의 제사용이다. 그렇기에 이걸 가장 잘 활용한 경우가 SEO 즉 검색엔진이다.   

## 컴포넌트의 종류:
컨테이너 형: 데이터, 컴포넌트 들을 가지고 있는 컴포넌트    

프레젠테이션 형: 컨테이너 형 컴포넌트 들을 출력하기 위한 용도의 컴포넌트  

이런 식으로 나눠서 관리를 한다면 좀 더 편하게 관리할 수 있다.   
리엑트의 이념과는 다르게 생각보다 재사용이 쉽지는 않다. 그러므로 나온 개념이 High Order Component(HOC)이다.
